Standard Operating Procedure (SOP)

Maintenance of AD Server, VMware ESXi & Datto Backup System

Prepared by: IT Team - Galaxy Freightline Inc. | Version 1.0

1. Purpose

To provide a standardized and safe procedure for rebooting and maintaining servers, including VMware ESXi hosts and Datto backup appliances, ensuring system stability, data integrity, and minimal downtime.

2. Scope

  • Physical servers
  • VMware ESXi Virtualization Platform
  • Datto Onsite & Cloud Backup Systems

3. Responsibilities

Applicable to all IT staff responsible for maintaining:

  • Perform scheduled maintenance, monitor logs, verify system backups.
  • Perform reboot/maintenance tasks, apply patches, verify system functionality
  • Validate application/service availability post-maintenance
  • Datto Onsite & Cloud Backup Systems

4. Maintenance Schedule Overview

System Daily Weekly Monthly Quarterly
Active Directory Check replication, logs Verify GPOs, DNS Windows updates Test restore
VMware ESXi Check host health Snapshot cleanup Firmware updates Patch review
Datto Backup Verify backup success Review alerts Test restore Firmware update

5. Procedures

5.1 Active Directory Server

  • Daily: Check Event Viewer, replication (repadmin /replsummary), DNS.
  • Weekly: Review GPOs, time sync, SYSVOL sharing.
  • Monthly: Apply Windows updates, run dcdiag, backup System State.
  • Quarterly: Test restore, audit user/group permissions.

5.2 VMware ESXi Host

  • Daily: Check host health, VM uptime.
  • Weekly: Clear snapshots, verify datastore space.
  • Monthly: Update VMware Tools, patch ESXi.
  • Quarterly: Validate vCenter compatibility, test restore.

5.3 Datto Backup

  • Daily: Confirm successful backup jobs.
  • Weekly: Check alerts, perform file restore test.
  • Monthly: Update agents, conduct full restore test.
  • Quarterly: Apply firmware updates, review retention policies.

6. Reporting & Documentation

  • Maintain a maintenance log for each task completed.
  • Submit monthly reports.
  • Log all incidents and resolutions in the helpdesk system.

7. Contingency & Escalation

Issue Action Escalation
